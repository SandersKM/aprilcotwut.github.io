# bayes_weighted_rating: calculates a bayes estimate weighted rating
#   based on an equation documented below (Question #2)   
def bayes_weighted_rating(df, this_col, m, C):
    v = df['counts']
    R = df[this_col]
    return ((v/(v+m)*R) + (m/(m+v)*C))
In [3]:
# new_review_overall: creates a new overall review score taking into
#   consideration all review categories which are weighted according
#   to their pearson correlation with review_overall
def new_review_overall(df, review_cols, corr):
    new_review = 0
    for col in review_cols:
        new_review = new_review + df[col]*corr[col]
    return new_review/sum(corr)

2. If you had to pick 3 beers to recommend using only this data, which would you pick?¶

For this, I'm going to recommend 3 "Top Beers". These beers are topping the charts, but likely not niche enough for your craft beer aficionado to be impressed. To achieve this, I decided to go with a weighted rating system utilizing a Bayes Estimator (like that used by IMDB movie ratings). Our weighted rating will be calculated with the following equation:

WR = (v*R)/(v+m) + (m*C)/(v+m)

WR: weighted rating
v: number of votes
m: minimum votes required to be listed
R: average rating of the beer as a number from 1 to 5
C: mean vote across the dataset

Additionally, rather than simply use the review_overall, I'm going to create a new "overall" rating which uses the Pearson Correlation to weight each section. We'll specifically use the following equation:

new_rating = sum(corr_i*r_i)/sum(corr_i) for all i

corr_i: correlation between rating overall and rating category 'i' r_i: rating for rating category 'i'

The value of this is in part because the overall is incredibly discrete rather than continuous - you must give your beer an overall rating between 1 and 5 at 0.5 increments. Implementing this, I hope to differentiate between users giving "high" 4.5 overall ratings and "low" 4.5 ratings (and similar).
